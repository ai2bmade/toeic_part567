# TOEIC RC Grammar Bot PRD

## 1. 제품 개요

Telegram에서 TOEIC RC 문법 학습을 매일 진행할 수 있는 공개 서비스용 봇을 만든다. 사용자는 Telegram user id 기준으로 식별되며, 허용된 사용자만 서비스에 접근할 수 있다.

서비스의 핵심은 매일 Part 5/6 기반 문법 학습 60문제를 풀고, 이를 완료한 유료 사용자에게만 Part 5/6/7 혼합 모의 퀴즈 10문제를 제공하는 것이다.

## 2. 제품 목표

- TOEIC RC 문법 유형을 매일 반복 학습하게 한다.
- Part 5뿐 아니라 요즘 TOEIC에 맞는 Part 6 문맥형 문법 문제도 학습에 포함한다.
- 하루 학습 60문제를 완료해야 모의 퀴즈를 볼 수 있게 하여 학습 지속성을 만든다.
- 오답 경험이 있는 유형을 이후 학습 문제에 섞어 반복 노출한다.
- 유료 사용자에게 하루 성과와 "오늘의 전국 랭킹"을 제공한다.

## 3. 제외 범위

- 단어 전용 무한 문제 모드
- 단어장 기능
- 듣기 LC 학습
- Part 7 일반 독해 학습 세트
- 실제 사용자 전체를 기반으로 한 실시간 랭킹
- 초기 버전의 웹 관리자 페이지

## 4. 사용자 구분

### 4.1 무료 사용자

- 하루 1세트, 총 6문제 이용 가능
- 무료 학습 세트는 Part 5에서만 출제
- 매일 00:00 KST에 무료 문제 수 리셋
- 하루 무료 세트를 완료하면 다음 메시지 표시

```text
오늘 무료 학습이 끝났습니다.
유료 등록 후 하루 60문제 학습 + 모의 퀴즈 10문제 이용이 가능합니다.
가입 방법은 추후 안내됩니다.
```

### 4.2 유료 사용자

- 하루 최대 70문제 이용 가능
- 학습 세트 60문제 + 모의 퀴즈 10문제
- 학습 60문제를 모두 완료해야 모의 퀴즈 진입 가능
- 유료 기간은 월간 기준
- 만료 1일 전 안내 메시지 발송
- 만료 후 자동으로 무료 사용자로 전환

## 5. 사용자 관리

사용자 정보는 Google Sheet 또는 CSV로 관리한다.

필수 필드:

```text
telegram_user_id
active
join_date
end_date
```

- `telegram_user_id`: Telegram user id
- `active`: 0 또는 1
- `join_date`: 유료 시작일
- `end_date`: 유료 종료일

접근 권한과 유료 여부는 이 사용자 리스트를 기준으로 판단한다.

## 6. 하루 기준

- 모든 일일 제한과 진행도는 KST 기준으로 계산한다.
- 매일 00:00 KST에 다음 항목을 리셋한다.
  - 무료 사용자 일일 문제 수
  - 유료 사용자 학습 세트 진행도
  - 유료 사용자 모의 퀴즈 응시 가능 여부
  - 당일 모의 퀴즈 결과
- 같은 날 중단 후 재접속하면 기존 학습 진행도에서 이어서 진행한다.
- 00:00 KST가 지나면 미완료 학습 진행도는 리셋된다.

## 7. 학습 세트

### 7.1 기본 구조

유료 사용자의 하루 학습량은 10세트, 총 60문제다. 무료 사용자는 하루 1세트, 총 6문제를 이용할 수 있다.

1세트는 다음 흐름으로 구성한다.

```text
대표 문제 1개
문법 개념 3줄 요약
유사 문제 2개
비유사 문제 3개
```

1세트는 총 6문제로 계산한다. 하루 최대 10세트까지 제공한다.

### 7.2 포함 파트

- 학습 세트에는 Part 5와 Part 6만 포함한다.
- 150개 문법 유형은 Part 5/6 문제에 매핑한다.
- Part 7은 학습 세트에 포함하지 않고 모의 퀴즈에서만 출제한다.

### 7.3 Part 6 카운트 방식

Part 6는 지문 단위가 아니라 문항 단위로 계산한다.

예:

```text
Part 6 지문 1개에 빈칸 문제가 2개 있으면 2문제로 카운트한다.
```

### 7.4 문법 유형 추천

하루 10세트의 문법 유형은 다음 방식으로 추천한다.

- 8세트: 사용자가 아직 풀어보지 않은 문법 카테고리에서 랜덤 추천
- 모든 카테고리를 이미 풀어봤다면 전체 카테고리에서 랜덤 추천
- 2세트: 사용자가 오답 경험이 있는 유형에서 랜덤 추천
- 오답 경험이 없으면 전체 카테고리에서 랜덤 추천

### 7.5 오답 반영

- 오답 복습은 별도 세션으로만 제공하지 않는다.
- 10세트를 푸는 중 비유사 문제 자리에 과거 오답과 유사한 문제를 섞는다.
- 하루 60문제를 완료하지 못한 경우에도 풀이 수와 오답 기록은 저장한다.
- 미완료 기록도 이후 약점 유형 추천에 반영한다.
- 단, 60문제를 완료하지 않으면 당일 모의 퀴즈 권한은 부여하지 않는다.

## 8. 모의 퀴즈

### 8.1 해금 조건

유료 사용자가 당일 학습 60문제를 완료해야 모의 퀴즈 10문제에 진입할 수 있다.

60문제 미완료 상태에서 모의 퀴즈를 요청하면 다음 취지의 안내를 표시한다.

```text
오늘 학습 60문제를 완료해야 모의 퀴즈를 볼 수 있습니다.
현재 진행도: n/60
```

### 8.2 구성

모의 퀴즈는 총 10문제로 구성한다.

```text
Part 5 단문 빈칸 완성: 3문제
Part 6 장문 빈칸 완성: 2문제
Part 7 Single Passage: 3문제
Part 7 Multiple Passage: 2문제
```

### 8.3 출제 방식

- 실제 TOEIC RC처럼 유형을 섞어서 출제한다.
- Part 5/6는 전체 150개 문법 유형 중 랜덤으로 출제한다.
- 같은 모의 퀴즈 안에서 동일 문법 유형이 중복되지 않도록 한다.
- Part 7은 모의 퀴즈에서만 사용한다.

## 9. 점수와 랭킹

### 9.1 점수

모의 퀴즈 종료 후 점수는 `맞힌 개수 / 10`으로 표시한다.

예:

```text
오늘의 점수: 8/10
```

### 9.2 랭킹 표시명

랭킹은 사용자에게 다음 이름으로 표시한다.

```text
오늘의 전국 랭킹
```

### 9.3 랭킹 숫자 산출

랭킹 표시용 숫자는 랜덤 integer로 산출한다.

```text
totalRankPool = 34275 이상 97211 이하의 랜덤 integer
displayRank = 정답 개수별 비율 구간에서 산출한 랜덤 integer
```

항상 다음 조건을 만족해야 한다.

```text
1 <= displayRank <= totalRankPool
```

정답 개수별 비율 구간:

```text
10개 정답: displayRank = totalRankPool
9개 정답: totalRankPool의 81%~87%
8개 정답: totalRankPool의 71%~80%
7개 정답: totalRankPool의 61%~70%
6개 정답: totalRankPool의 51%~70%
5개 정답: totalRankPool의 31%~50%
4개 정답: totalRankPool의 26%~30%
3개 정답: totalRankPool의 21%~25%
2개 정답: totalRankPool의 11%~20%
1개 정답: totalRankPool의 3%~10%
0개 정답: totalRankPool의 1%~2%
```

구현 시 `floor` 또는 `ceil`을 사용해 integer 범위를 만들고, 마지막에 반드시 `displayRank <= totalRankPool` 보정을 적용한다.

표시 예:

```text
오늘의 전국 랭킹: 39,678 / 45,678
```

## 10. 문제 형식

### 10.1 Part 5

- 단문 빈칸 완성형
- 보기 A/B/C/D 4개
- 문법 유형 또는 문맥형 문법 판단을 포함
- 무료 학습 세트는 Part 5에서만 출제

### 10.2 Part 6

- 장문 빈칸 완성형
- 문법, 어휘 문맥, 문장 흐름 판단을 포함할 수 있음
- 학습 세트와 모의 퀴즈에 포함
- 문항 단위로 카운트

### 10.3 Part 7

- Single Passage와 Multiple Passage를 포함
- 학습 세트에는 포함하지 않음
- 모의 퀴즈에서만 출제

## 11. 문법 개념 설명

대표 문제 풀이 후 문법 개념은 3줄 요약으로 제공한다.

요건:

- 정답 근거를 짧게 설명
- 자주 나오는 함정 포인트 포함
- 필요하면 핵심 단어 뜻은 해설 안에 포함
- 별도 단어장 기능은 만들지 않음

## 12. 문제 데이터 관리

### 12.1 문제 마스터

문제는 Google Sheet를 마스터 DB로 관리하는 것을 권장한다. 봇은 초기에는 CSV export를 읽고, 운영 버전에서는 Google Sheet API 연동을 고려한다.

추천 시트:

```text
users
grammar_types
questions
daily_records
wrong_answers
mock_results
```

### 12.2 questions 필드 후보

```text
question_id
part
grammar_type_id
grammar_type_name
passage_id
passage_text
question_text
choice_a
choice_b
choice_c
choice_d
correct_choice
explanation
similarity_group
status
review_note
created_at
updated_at
```

`status` 값:

```text
draft
reviewing
approved
rejected
```

봇은 `status = approved`인 문제만 출제한다.

## 13. AI 문제 생성 및 검수

문제 생성은 이 Codex 쓰레드에서 진행한다.

운영 흐름:

1. 사용자가 이 쓰레드에 "문제 만들자"라고 요청한다.
2. 사용자가 문법 유형 샘플과 보기 4개의 옵션 유형을 제공한다.
3. AI가 해당 유형에 대해 30문제를 생성한다.
4. 생성 결과에는 다음을 포함한다.
   - 유형
   - 문제
   - 보기 A/B/C/D
   - 정답
   - 해설
5. 사용자가 문제를 수정하고 검수한다.
6. 승인된 문제를 Google Sheet의 `questions` 시트에 업로드한다.
7. 봇은 approved 문제만 사용한다.

## 14. AI 사용 및 비용 원칙

학생이 문제를 푸는 실시간 학습 과정에서는 OpenAI API를 호출하지 않는다.

운영 원칙:

- 문제, 보기, 정답, 해설은 사전에 AI로 생성한다.
- 운영자가 생성된 문제를 검수하고 수정한다.
- 검수 완료된 문제만 DB에 저장한다.
- 봇은 DB에 저장된 `approved` 문제만 출제한다.
- 학생 학습 중에는 DB 출제, 채점, 기록, 랭킹 계산만 수행한다.
- 따라서 학생 1명이 하루 70문제를 모두 풀어도 OpenAI 토큰 비용은 발생하지 않는다.
- OpenAI 토큰 비용은 운영자가 "문제 만들자" 흐름으로 신규 문제를 생성할 때만 발생한다.

이 구조를 통해 학생 수가 늘어나도 AI 비용이 학생 수에 비례해 증가하지 않도록 한다. 서비스는 저렴한 가격으로 제공할 수 있어야 한다.

## 16. 성공 기준

- 무료 사용자가 하루 1세트, 총 6개의 Part 5 문제를 풀 수 있다.
- 유료 사용자가 하루 10세트, 총 60문제를 이어서 풀 수 있다.
- 같은 날 중단 후 재접속하면 진행도가 유지된다.
- 00:00 KST가 지나면 일일 진행도가 리셋된다.
- 60문제를 완료한 유료 사용자만 모의 퀴즈 10문제를 볼 수 있다.
- 모의 퀴즈는 Part 5/6/7 구성 비율을 지킨다.
- 모의 퀴즈 종료 후 점수와 오늘의 전국 랭킹을 표시한다.
- 랭킹 숫자는 항상 integer이며 전체 숫자를 넘지 않는다.
- 유료 만료 1일 전 안내가 가능하다.
- 유료 만료 후 자동으로 무료 사용자로 전환된다.
- 학생 학습 중 OpenAI API 호출 없이 모든 문제 출제와 채점이 가능하다.

- 유료 가입 안내 문구
- Part 7 문제 데이터 생성 및 검수 방식
